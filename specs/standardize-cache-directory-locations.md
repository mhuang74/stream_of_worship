## 1. Executive Summary

This specification addresses the inconsistency in cache directory locations across different components of the Stream of Worship system. Currently, cache files (audio, stems, lyrics) are stored in different locations depending on which command or component generates them, leading to:

- Duplicate downloads of the same assets
- Confusion about where cached files are located
- Inefficient disk usage
- Broken user expectations when switching between `sow-admin` and `sow-app`

**Goal**: Unify all cache directory locations to a single, configurable location used consistently across all components.

## 2. Problem Statement

### 2.1 Current State

| Component/Command | Cache Directory Location | Example Path |
|-------------------|-------------------------|--------------|
| `sow-admin audio vocal` | `{db_path.parent}/cache` | `~/.local/share/sow-admin/cache/` |
| `sow-admin audio cache` | Hardcoded to `~/.config/sow-app/cache` | `~/.config/sow-app/cache/` |
| `sow-app` TUI | `get_app_config_dir() / "cache"` | `~/.local/share/sow-app/cache/` |
| `sow-admin audio playback` | Hardcoded to `~/.config/sow-app/cache` | `~/.config/sow-app/cache/` |

### 2.2 Impact

1. **Duplicate Storage**: Same audio file downloaded by `sow-admin audio cache` and `sow-app` are stored in different locations
2. **Inconsistent Behavior**: User expects `vocals_clean.wav` generated by `audio vocal` to be available to `audio cache` command
3. **Configuration Fragmentation**: Cache location configuration exists in multiple places with different defaults
4. **Maintenance Burden**: Developers must remember which component uses which cache location

## 3. Desired State

### 3.1 Unified Cache Location Strategy

All components should use a **single, centralized cache directory** determined by:

1. **Environment variable** `SOW_CACHE_DIR` (highest priority)
2. **Configuration file** setting `cache_dir` (if exists)
3. **Platform-appropriate default** (lowest priority):
   - Linux: `~/.cache/sow/` or `~/.local/share/sow/cache/`
   - macOS: `~/Library/Caches/sow/`
   - Windows: `%LOCALAPPDATA%\sow\cache\`

### 3.2 Directory Structure


## 4. Implementation Design

### 4.1 Core Changes

#### 4.1.1 Create Centralized Cache Path Resolution

**New File**: `src/stream_of_worship/core/cache_paths.py`

```python
"""Centralized cache path resolution for all SOW components.

This module provides a single source of truth for cache directory locations,
ensuring consistency across sow-admin, sow-app, and other components.
"""

import os
from pathlib import Path
from typing import Optional


def get_sow_cache_dir(
    env_var: str = "SOW_CACHE_DIR",
    config_path: Optional[Path] = None,
) -> Path:
    """Get the unified SOW cache directory.
    
    Resolution order (highest to lowest priority):
    1. Environment variable SOW_CACHE_DIR
    2. Configuration file cache_dir setting (if provided)
    3. Platform-appropriate default
    
    Args:
        env_var: Name of environment variable to check
        config_path: Optional path to config file to read from
        
    Returns:
        Path to cache directory (creates if doesn't exist)
        
    Examples:
        >>> get_sow_cache_dir()  # Uses default location
        Path('/home/user/.cache/sow')
        
        >>> os.environ['SOW_CACHE_DIR'] = '/tmp/sow-cache'
        >>> get_sow_cache_dir()
        Path('/tmp/sow-cache')
    """
    # Priority 1: Environment variable
    if env_value := os.environ.get(env_var):
        cache_dir = Path(env_value)
        cache_dir.mkdir(parents=True, exist_ok=True)
        return cache_dir
    
    # Priority 2: Config file (if provided and exists)
    if config_path and config_path.exists():
        # Read from config file
        cache_dir = _read_cache_dir_from_config(config_path)
        if cache_dir:
            cache_dir.mkdir(parents=True, exist_ok=True)
            return cache_dir
    
    # Priority 3: Platform default
    cache_dir = _get_platform_default_cache_dir()
    cache_dir.mkdir(parents=True, exist_ok=True)
    return cache_dir


def _get_platform_default_cache_dir() -> Path:
    """Get platform-appropriate default cache directory."""
    import sys
    
    if sys.platform == "darwin":
        # macOS: ~/Library/Caches/sow/
        return Path.home() / "Library" / "Caches" / "sow"
    elif sys.platform == "win32":
        # Windows: %LOCALAPPDATA%\sow\cache\
        local_app_data = os.environ.get("LOCALAPPDATA", Path.home() / "AppData" / "Local")
        return Path(local_app_data) / "sow" / "cache"
    else:
        # Linux/Unix: ~/.cache/sow/ (XDG spec)
        xdg_cache = os.environ.get("XDG_CACHE_HOME", Path.home() / ".cache")
        return Path(xdg_cache) / "sow"


def _read_cache_dir_from_config(config_path: Path) -> Optional[Path]:
    """Read cache_dir from config file."""
    try:
        import json
        with open(config_path) as f:
            config = json.load(f)
        if cache_dir_str := config.get("cache_dir"):
            return Path(cache_dir_str)
    except (json.JSONDecodeError, KeyError, IOError):
        pass
    return None


def get_recording_cache_path(
    hash_prefix: str,
    cache_dir: Optional[Path] = None,
) -> Path:
    """Get cache directory for a specific recording.
    
    Args:
        hash_prefix: Recording hash prefix
        cache_dir: Optional override cache directory
        
    Returns:
        Path to recording's cache directory
    """
    base = cache_dir or get_sow_cache_dir()
    return base / hash_prefix
```

### 4.3 Migration Strategy

#### 4.3.1 Backward Compatibility

To avoid breaking existing installations:

1. **Check for existing cache directories** at old locations
2. **Migrate or warn** about existing caches
3. **Symlink option** (Unix only): Create symlinks from old to new location

**Migration Utility** (optional):
```python
def migrate_legacy_caches(
    new_cache_dir: Path,
    old_locations: list[Path],
    dry_run: bool = True,
) -> dict[Path, Path]:
    """Migrate or report legacy cache directories.
    
    Args:
        new_cache_dir: New unified cache location
        old_locations: List of legacy cache directories to check
        dry_run: If True, only report what would be moved
        
    Returns:
        Dictionary mapping old paths to new paths
    """
    migrations = {}
    
    for old_dir in old_locations:
        if not old_dir.exists():
            continue
            
        # For each hash_prefix in old location
        for hash_dir in old_dir.iterdir():
            if not hash_dir.is_dir():
                continue
                
            new_hash_dir = new_cache_dir / hash_dir.name
            
            if dry_run:
                print(f"Would migrate: {hash_dir} -> {new_hash_dir}")
            else:
                # Move or merge directories
                if new_hash_dir.exists():
                    # Merge: copy files that don't exist in new location
                    _merge_cache_dirs(hash_dir, new_hash_dir)
                else:
                    shutil.move(str(hash_dir), str(new_hash_dir))
                    
            migrations[hash_dir] = new_hash_dir
            
    return migrations
```

#### 4.3.2 Configuration Migration

Update existing config files to use new default:

```python
def migrate_config_cache_setting(config_path: Path) -> bool:
    """Update config file to use new cache location if using legacy default.
    
    Args:
        config_path: Path to config file
        
    Returns:
        True if migration occurred
    """
    if not config_path.exists():
        return False
        
    import json
    
    with open(config_path) as f:
        config = json.load(f)
    
    # Check if using old hardcoded path
    old_paths = [
        str(Path.home() / ".config" / "sow-app" / "cache"),
        str(Path.home() / ".local" / "share" / "sow-app" / "cache"),
    ]
    
    current_cache = config.get("cache_dir", "")
    if current_cache in old_paths:
        # Remove explicit setting to use new default
        del config["cache_dir"]
        
        with open(config_path, "w") as f:
            json.dump(config, f, indent=2)
        return True
        
    return False
```

```python
        """Environment variable should take highest priority."""
        env_dir = tmp_path / "env-cache"
        with patch.dict(os.environ, {"SOW_CACHE_DIR": str(env_dir)}):

        """Config file should be used if env var not set."""
        config_dir = tmp_path / "config-cache"

        config_file.write_text(f'{{"cache_dir": "{config_dir}"}}')
        
        with patch.dict(os.environ, {}, clear=True):
            result = get_sow_cache_dir(config_path=config_file)
            assert result == config_dir

        """Platform default should be used if nothing else set."""
        with patch.dict(os.environ, {}, clear=True):
            result = get_sow_cache_dir(config_path=tmp_path / "nonexistent.json")
            # Should return platform-appropriate path
            assert result.name == "sow"
```

## 9. Open Questions

1. **Should we migrate existing caches automatically or just warn?**
   - Auto-migration: Better UX, risk of data loss
   - Warning only: Safer, requires user action
   - Recommended: Warning + opt-in migration flag

2. **Should temporary processing directories be cleaned up?**
   - Stage 1/2 files in `vocal_extraction/` are intermediate
   - Pros of cleanup: Save disk space
   - Cons of cleanup: Can't resume interrupted processing
   - Recommended: Clean up on success, keep on failure for debugging

3. **Should we add `--cache-dir` CLI option to all commands?**
   - Pros: Flexibility for testing, multiple cache scenarios
   - Cons: More flags to maintain, potential confusion
   - Recommended: Add to major commands (cache, vocal, playback)
